### ● フェーズの学習目標

## note

## f文字列（f-string）とは

文字列の中に**変数を直接埋め込める**書き方です。

```python
name = "Community Garden"
print(f"Garden: {name}")
# 出力: Garden: Community Garden
```

頭に `f` をつけて、変数は `{}` で囲むだけです。

---

## `+` でつなぐ方法との比較

```python
name = "Community Garden"
age = 5

# + でつなぐ方法（古い書き方）
print("Garden: " + name + " Age: " + str(age))

# f文字列（スッキリ！）
print(f"Garden: {name} Age: {age}")
```

`+` でつなぐ場合、数字は `str()` で文字列に変換しないとエラーになりますが、f文字列なら**そのまま書けます**。

---

## `{}` の中では計算もできる

```python
length = 5
width = 3
print(f"Plot area: {length * width}")
# 出力: Plot area: 15
```

---

## まとめ

| 方法 | 書き方 | 特徴 |
|---|---|---|
| `+` 連結 | `"Hello " + name` | シンプルだが長くなりがち |
| f文字列 | `f"Hello {name}"` | 読みやすくておすすめ |



## 何が起きているか

```python
age = 5
f"Age: {age}"
```

これは実際には**別々のパーツを組み合わせて新しい文字列を作っています。**

```
"Age: "  +  str(5)  →  "Age: 5"  （新しいstr）
```

---

## イメージ図

```
f"Age: {age}"
     ↓
"Age: " ← これはstr
  {age} ← これはintだけど str()に変換される
     ↓
"Age: 5" ← 全部合わさって新しいstrが完成
```

---

## 変数自体の型は変わらない

```python
age = 5
text = f"Age: {age}"

print(type(age))   # <class 'int'>  ← ageはintのまま
print(type(text))  # <class 'str'>  ← textはstr
```

`age` 自体はintのまま、**f文字列が新しいstrを作る**だけです。

---

## Cで例えると

```c
int age = 5;
printf("Age: %d", age);  // ageはintのまま、printfが表示を変換
```

Cの `printf` が型を変換して表示するのと似た感覚です。`age` 自体の型は変わらないですよね。それと同じです！
